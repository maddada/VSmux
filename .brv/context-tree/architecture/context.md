# Domain: architecture

## Purpose

Contains architectural knowledge about the terminal workspace frontend, workspace projection logic, and backend daemon behavior for VSmux terminal sessions.

## Scope

Included in this domain:

- Terminal workspace rendering and pane lifecycle
- Runtime caching and visibility behavior
- Workspace message handling and pane ordering
- Backend daemon, leases, and persisted session state behavior

Excluded from this domain:

- Unrelated editor features outside terminal workspace
- Generic terminal usage instructions

## Ownership

VSmux terminal workspace implementation

## Usage

Use this domain for current architectural state and behavioral contracts of the terminal workspace stack.
